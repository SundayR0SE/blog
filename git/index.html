                            # 思维导图

                            # 常见名词

                            Workspace：工作区
                            Index / Stage：暂存区
                            Repository：仓库区（或本地仓库）
                            Remote：远程仓库

                            # 安装

                            GitHub for Windows
                            GitHub for Mac

                            # 配置

                            设置提交代码时的用户信息
                            $ git config --global user.name "[name]"
                            $ git config --global user.email "[email address]"

                            # 我常用的工作流程

                            • 在Githup/Gitee新建一个仓库,然后在本地 $ git clone url
                            • 新建一个myfeature分支 $ git checkout -b myfeature
                            • 写完代码后，提交到暂存区 $ git add .
                            • 将暂存区的提交到本地仓库 $ git commit -m 'message'
                            • 提交到线上仓库 $ git push
                            • 切换到主分支 $ git checkout master
                            • 主分支合并分支 $ git merge myfeature
                            • 切换回myfeature分支开发 $ git checkout myfeature
                            • 从远程仓库拉取代码 $ git pull [remote] [branch]

                            # 一. 新建代码库

                            • 在当前目录新建一个Git代码库$ git init
                            • 新建一个目录，将其初始化为Git代码库$ git init [project-name]
                            • 下载一个项目和它的整个代码历史$ git clone [url]

                            # 二. 提交到暂存区

                            • 添加当前目录的所有文件到暂存区$ git add .

                            # 三. 代码提交

                            • 提交暂存区到仓库区$ git commit -m [message]

                            # 四. 分支

                            • 列出所有本地分支和远程分支$ git branch -a
                            • 新建一个分支，但依然停留在当前分支$ git branch [branch-name]
                            • 新建一个分支，并切换到该分支$ git checkout -b [branch]
                            • 新建一个分支，指向指定commit$ git branch [branch] [commit]
                            • 新建一个分支，与指定的远程分支建立追踪关系$ git branch --track [branch] [remote-branch]
                            • 切换到指定分支，并更新工作区$ git checkout [branch-name]
                            • 切换到上一个分支$ git checkout -
                            • 建立追踪关系，在现有分支与指定的远程分支之间$ git branch --set-upstream [branch] [remote-branch]
                            • 合并指定分支到当前分支$ git merge [branch]
                            • 删除分支$ git branch -d [branch-name]
                            • 删除远程分支$ git branch -dr [remote/branch]

                            # 五. 查看信息

                            • 显示有变更的文件$ git status
                            • 显示当前分支的版本历史$ git log
                            • 显示暂存区和工作区的差异$ git diff

                            # 六. 远程同步

                            • 下载远程仓库的所有变动$ git fetch [remote]
                            • 取回远程仓库的变化，并与本地分支合并$ git pull [remote] [branch]
                            • 上传本地指定分支到远程仓库$ git push [remote] [branch]
                            • 强行推送当前分支到远程仓库，即使有冲突$ git push [remote] --force

                            # 七. 撤销

                            • 恢复暂存区的指定文件到工作区$ git checkout [file]
                            • 恢复某个commit的指定文件到暂存区和工作区$ git checkout [commit] [file]
                            • 恢复暂存区的所有文件到工作区$ git checkout .
                            • 重置暂存区的指定文件，与上一次commit保持一致，但工作区不变$ git reset [file]
                            • 回退到上一个版本$ git reset --hard HEAD^
                            • 回退到指定版本$ git reset --hard 48e56e
